## 🏗️ АРХИТЕКТУРНАЯ ПРОБЛЕМА

### Текущая структура (МОНОЛИТ)
```
src/
├── features/       # UI + logic (всё смешано!)
├── pages/          # Толстые компоненты
├── shared/         # Utils + бизнес-логика
└── (нет domain/, data/, integration/, state/)
```

### Целевая структура (MODULAR)
```
src/
├── domain/         # Чистая бизнес-логика (types, rules, calculations)
├── data/           # Repositories и кэширование
├── integration/    # Обёртки над API (OpenFoodFacts, AI и т.д.)
├── state/          # Тонкий Redux (только UI-state)
├── features/       # Use cases + hooks + компоненты
└── shared/         # Только утилиты
```

### Почему это важно
- 🧹 Код организован по слоям
- 📝 Unit-тесты легко писать
- 🔄 Логику можно переиспользовать
- 📈 Масштабирование без боли
- 🔗 Интеграции подключаются легко
